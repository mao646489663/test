Permissions Summary

    The summary page displays an overview of permissions currently set and allows you to update and remove permissions for user types (guests, registered users), users and groups.

    To grant a new permission, click on the "Grant New Permission" tab. If you just want to update permissions, use the main summary page. Click on the additional permission you want to grant and click "Save Changes." To remove a permission, click on the delete icon in the row associated with a user type, user or group.

    Note, it is not possible to completely remove the "Anyone" or "Registered Users" types rows. To remove all permissions for these types, click on the remove icon. To grant new permissions for these types, click on the desired permissions and click "Save Changes."

Grant Permissions To

    Permissions can be granted to users or groups, or two special classes of users:

    • "Anybody" (Guests)

      Anybody visiting the site matches the "guest" permission, including users that haven't logged-in.

    • "Registered Users"

      "Registered Users" are anybody that has a user account and is logged in. One common permission setup is to allow creation of threads and messages by registered users, and read permissions for guests.

    • User

      A permission for a particular user. Enter the user's name.

    • Group

      A permission for a particular group. Enter the group's name.

Admin Permissions

    Administrative permissions in Jive Forums define a distinct set of roles that can be used to divide administrative duties. Note: only Jive Forums Professional and Enterprise support role-based administration. Jive Forums Lite only allows a single system administrator.

    • "System Admin"

      System administrators have complete control of the system. They are the only role permitted to use panels on the Global Settings tab.

    • "Category Admin"

      Category administrators can create and delete sub-categories and forums inside the category they are admins for. Additionally, they have the same abilities as forum administrators for all forums in the category and sub-categories.

    • "Forum Admin"

      A forum administrator controls all settings for the forum and manages the forum's content.

    • "Moderator"

      Moderators have the ability to manage content in the forums for which they are a moderator. Additionally, if "moderation" is turned on, they have the ability to reject or accept threads and messages.

Content-based Permissions
    Content permissions control who is allowed to read and create content.

    • "Read Forum"
      Grants the ability to read forum content.

    • "Create Thread"
      Grants the ability to create new threads. The ability to create threads automatically means users are able to create messages in existing threads.

    • "Create Message"
      Grants the ability to create new messages.

    • "Create Attachment"
      Grants the ability to add file attachments to messages. Note: attachments are only supported in Jive Forums Professional and Enterprise.
